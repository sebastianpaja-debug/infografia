Metodos de Organizacion

Estrategias que el SO emplea para distribuir los bloques de un archivo en el disco.

Asignacion contigua

Los bloques del archivo se almacenan de forma consecutiva. Ofrece acceso rapido, pero genera fragmentacion externa al eliminar archivos.

A1 A2 A3 A4 — — B1 B2

Bloques contiguos del archivo A, espacio libre, bloques del archivo B.

Asignacion enlazada

Cada bloque contiene un puntero al siguiente, formando una lista enlazada. Elimina la fragmentacion externa, pero el acceso secuencial es mas lento.

A1 → — → A2 → A3 → — → A4

Punteros enlazan bloques no contiguos del archivo A.

Asignacion indexada

Un bloque indice contiene punteros a todos los bloques de datos del archivo. Combina acceso directo con flexibilidad de ubicacion.

IDX → D1 D2 D3 D4

El bloque indice (IDX) apunta a los bloques de datos D1-D4.

Tabla de asignacion de archivos (FAT)

Una tabla centralizada al inicio del volumen registra la cadena de bloques de cada archivo. Usada en FAT12, FAT16 y FAT32.

FAT → C1 → C2 → C3

La tabla FAT contiene las cadenas de bloques para cada archivo.
